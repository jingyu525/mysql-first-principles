# 学习路线图

## 为什么这样学？

大多数 MySQL 教程的问题在于：

```text
直接讲索引 → 直接讲事务 → 直接讲 MVCC
```

但它们都建立在一堆**没有说出来的假设**上。如果这些假设没建立起来：索引=死记硬背、事务=死记硬背、MVCC=死记硬背。

本教程的路线是：**从"为什么存在"到"如何实现"**。

---

## 完整路线

### 第一阶段：数据库世界观

```text
数据库是什么 → 表是什么 → 主键 → 索引 → B+Tree → 事务 → MVCC（概览）
```

**目标**：建立对 MySQL 的全局认知，知道每个概念在整体中的位置。

**对应章节**：[第一章](ch01-worldview/README.md)

---

### 第二阶段：从数据结构到 B+Tree

```text
数组 → 链表 → 二叉搜索树 → 平衡树 → B树 → B+Tree
```

**目标**：理解"为什么 B+Tree 是数据库索引的最优选择"，而不是只会背结论。

**对应章节**：[第二章](ch02-data-structure/README.md)

---

### 第三阶段：存储引擎基础

```text
磁盘 → Page → Buffer Pool → B+Tree → 聚簇索引 → 二级索引 → 回表
```

**目标**：理解"数据库以 Page 为工作单位"，这是所有性能分析的物理基础。

**对应章节**：[第三章](ch03-storage-engine/README.md)

---

### 第四阶段：索引体系

```text
聚簇索引 → 二级索引 → 联合索引 → 最左匹配 → 覆盖索引 → 索引失效 → WHERE/GROUP BY/ORDER BY索引联动
```

**目标**：能够设计索引、分析索引命中情况、理解 90% 慢 SQL 的根因，掌握索引如何服务完整 SQL 流水线。

**对应章节**：[第四章](ch04-index/README.md)

---

### 第五阶段：事务与持久化

```text
事务状态机 → ACID → Buffer Pool 风险 → Redo Log → Undo Log → Crash Recovery
```

**目标**：理解"数据库如何保证数据不会丢"，掌握 WAL 核心思想。

**对应章节**：[第五章](ch05-transaction/README.md)

---

### 第六阶段：MVCC 与并发读

```text
读写冲突 → 版本化 → Undo 链 → Read View → 隔离级别
```

**目标**：理解"数据库如何在高并发下同时保证性能和正确性"。

**对应章节**：[第六章](ch06-mvcc/README.md)

---

### 第七阶段：锁

```text
写写冲突 → Record Lock → Gap Lock → Next-Key Lock → 死锁
```

**目标**：理解 MVCC 的边界，掌握锁的类型和死锁机制。

**对应章节**：[第七章](ch07-lock/README.md)

---

### 第八阶段：执行计划

```text
SQL → Parser → Optimizer → Executor → EXPLAIN
```

**目标**：能够用 EXPLAIN 分析任何 SQL 的执行路径，定位性能瓶颈。

**对应章节**：[第八章](ch08-explain/README.md)

---

### 第九阶段：InnoDB 内核

```text
Page → Extent → Segment → Buffer Pool → Dirty Page → Flush → Checkpoint → Crash Recovery
```

**目标**：把 MySQL 看成一个**管理磁盘数据的操作系统**，建立完整的系统视角。

**对应章节**：[第九章](ch09-innodb/README.md)

---

### 延伸阅读：架构演化

```text
业务增长 → 数据增长 → 资源瓶颈 → 架构演化 → 新问题 → 继续演化
```

**目标**：建立架构师视角——理解分库分表不是目标，系统演化能力才是。掌握单库单表 → 缓存 → 读写分离 → 分区 → 分库分表 → 分布式数据库的完整演化路线和决策框架。

**对应章节**：[附录：架构演化](appendix/architecture-evolution.md)

---

## 学习建议

### 如果你是从零开始

按第一阶段 → 第九阶段的顺序，每章大约需要 1-2 小时阅读 + 实验。

建议的学习节奏：

| 周次 | 内容 |
|------|------|
| 第 1 周 | 第一、二章（世界观 + 数据结构→B+树） |
| 第 2 周 | 第三、四章（存储引擎 + 索引体系） |
| 第 3 周 | 第五、六章（事务 + MVCC） |
| 第 4 周 | 第七、八章（锁 + EXPLAIN） |
| 第 5 周 | 第九章（InnoDB 内核） |

### 如果你是有经验的后端工程师

可以从第三、四章开始精读，然后跳到第五、六、七章。第一、二章可以快速浏览。

### 如果你正在准备面试

重点阅读：第四章（索引体系）、第五章（事务）、第六章（MVCC）、第七章（锁）。

---

## 核心认知转变

```text
学习前：
    MySQL = SQL 语法 + 索引 + 事务（一堆孤立的知识点）

学习后：
    MySQL = 一个专门解决海量数据存储、检索、一致性和并发控制的大型运行时系统
    InnoDB = 管理磁盘数据的"操作系统内核"
```
