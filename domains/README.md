# 领域视角

> 这是 MySQL 第一性原理的**第二种阅读路径**。与推导链视角（ch01→ch09，纵向因果递进）不同，领域视角按功能域横切，适合按模块查漏补缺。

---

## 两套视角对比

| | 推导链视角（ch01~ch09） | 领域视角（domains/） |
|---|---|---|
| **读法** | 从头到尾，因果递进 | 按兴趣跳读，模块化 |
| **适合** | 第一次系统学习 | 复习 / 查漏补缺 / 面试准备 |
| **组织** | 数据结构→存储→索引→事务→... | 存储篇 / 事务篇 / 并发篇 / ... |
| **内容** | 完整正文 + 推导过程 | 知识地图 + 跨章链接 + 领域总结 |

## 推荐阅读路径

```
如果你是初学者：
  推导链 ch01 → ch02 → ... → ch09（按顺序）
  然后回到这里，用领域视角做全局回顾

如果你已经有基础：
  从感兴趣的领域开始（如 [并发篇] → [事务篇] → [存储篇]）
  用知识地图 + 跨章链接快速定位到正文

如果你是面试准备：
  每个领域的「核心问题」+「知识地图」= 面试提纲
  快速过一遍，对不熟的点击链接深入
```
